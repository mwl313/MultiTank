## 1. 게임 정체성

### 장르
2D 탑다운 스플릿 컨트롤 탱크 서바이벌 (뱀서라이크 파생)

### 코어 컨셉
**"같은 탱크, 다른 역할"**

두 명의 플레이어가 하나의 탱크를 조종한다.
- **운전수:** 탱크 이동, 회전, 드리프트, 전투 기동
- **포수 (사격수):** 포탑 조준, 무기 선택/발사, 화력 집중

서로의 역할이 완전히 다르기 때문에 필연적인 소통이 발생하고, 그것이 핵심 재미.

### 필수 사양
| 항목 | 결정 |
|:-----|:-----|
| 멀티플레이 | **2인 1조 온라인 협동 (목표 사양)** — WebSocket 구현 예정 |
| 시점 | 탑다운 2D (카메라가 탱크 추적) |
| 조작 | 운전수: WASD + 액션키 / 포수: 마우스 조준+클릭 + 숫자키 무기선택 |
| 솔로 | MVP는 통합 조작으로 1인이 두 역할 수행 (멀티 스텁 유지) |
| 패배 조건 | 탱크 HP 0 |

## 2. 3대 차별점 (컨셉 — 세부 설계는 추후)

> `05-게임디자인-뼈대.md` v0.3의 차별점 컨셉을 유지하되, 세부 메커니즘(스킬트리, 세이프존 등)은 백지화되어 별도 기획 예정.

### ① 역할 분리 기반 협동
- 운전수와 포수는 서로 다른 조작 방식, 다른 게임플레이 목표
- 필연적인 소통과 전략적 결정 필요
- 솔로 모드에서도 통합 조작으로 두 역할의 재미를 모두 경험 가능

### ② 운전수도 전투 기여
- 운전수는 단순 이동 담당이 아니라 직접 전투에 기여
- 현재: 드리프트 시 측면 충돌 적에게 데미지 (mvp-v2 구현 완료)
- 추후: 운전수 전용 액션/성장 시스템 별도 기획 예정

### ③ 전장 이동이 게임플레이
- 기존 뱀서라이크의 정적 버티기와 달리 탱크가 전장을 누비며 싸움
- 이동 자체가 게임플레이 — 운전수의 기동력이 핵심 자원
- 추후: 게임 진행 시스템(목표/씬/웨이브) 별도 기획 예정

## 3. 조작 & 입력 시스템

### 3.1 운전수 (Driver)

| 조작 | 기능 |
|:----|:-----|
| **W** | 전진 가속 |
| **S** | 후진 감속 |
| **A** | 좌회전 (차체 방향 기준) |
| **D** | 우회전 (차체 방향 기준) |
| **W+A/D** 유지 | 드리프트 (전진 중 선회 시 측면 충돌 데미지) |

> 탱크는 velocity 벡터 기반 이동. 입력 즉시 방향 전환이 아닌 가속/마찰 물리.

### 3.2 포수 — 사격수 (Gunner)

| 조작 | 기능 |
|:----|:-----|
| **마우스 이동** | 포탑 조준 (차체 방향과 독립) |
| **마우스 좌클릭** | 현재 선택된 무기 발사 (쿨다운 적용) |
| **숫자키 1** | 💥 폭발 무기 선택 |
| **숫자키 2** | 🔫 속사 무기 선택 |
| **숫자키 3** | 🎯 저격 무기 선택 |
| **숫자키 4** | 🔩 관통 무기 선택 |
| **숫자키 동시 (1+2 등)** | 융합 무기 선택 (100ms 이내 동시 입력 판정) |

> 무기 시스템 상세: §5 사격수 무기 시스템 참조.

### 3.3 공통

| 조작 | 기능 | 씬 |
|:----|:-----|:---|
| **R** | 게임 리스타트 | 게임 오버/플레이 |
| **ESC** | 일시정지 / 메뉴 | 플레이 |
| **Enter / Click** | 다음 화면 진행 | 메뉴/결과 |
| **Tab** | (추후) 업그레이드 UI | 플레이 |

## 5. 사격수 무기 시스템

> 이 섹션은 `07-사격수-무기-시스템.md`의 전체 내용을 기획서에 통합한 것입니다.
> 원본 문서의 모든 설계가 그대로 적용되며, 여기서는 요약만 포함합니다.
> 상세 수치, 테이블, 구현 가이드는 `07-사격수-무기-시스템.md` 원문 참조.

### 5.1 개요
사격수는 4종 단일 무기와 6종 융합 무기를 사용.
각 무기는 독립적인 능력치, 탄창, 재장전 시간을 가짐.
랜덤 가챠 기반 강화 시스템으로 최대 20단계까지 성장.

### 5.2 단일 무기 (4종)

| ID | 이름 | 아이콘 | 코어 컨셉 |
|:--:|:-----|:------:|:---------|
| 1 | 💥 폭발 (Explosion) | 💥 | 탄착 지점 넓은 범위 피해 |
| 2 | 🔫 속사 (Rapid Fire) | 🔫 | 1회 발사 시 여러 발 포탄 |
| 3 | 🎯 저격 (Sniper) | 🎯 | 3배 사거리, 거리 비례 데미지 |
| 4 | 🔩 관통 (Piercing) | 🔩 | 적/지형 관통, 관통 시 데미지 감소 |

### 5.3 융합 무기 (6종)

| 조합 | 코드 | 데미지 규칙 |
|:----|:-----|:------------|
| 폭발 + 속사 | 1+2 | 폭발 데미지, 속사 발사 간격 |
| 폭발 + 저격 | 1+3 | 폭발 데미지, 원거리 시 저격 계수 |
| 폭발 + 관통 | 1+4 | 관통 중엔 폭발 없음, 최종 탄착지 폭발 |
| 속사 + 저격 | 2+3 | 높은 데미지 쪽, 원거리 시 저격 계수 |
| 속사 + 관통 | 2+4 | 높은 데미지 쪽 |
| 저격 + 관통 | 3+4 | 높은 데미지 쪽, 원거리 시 저격 계수 |

### 5.4 공용 옵션
- **데미지:** 적에게 입히는 피해
- **탄창 수:** 한 탄창 내 포탄 수
- **재장전 시간:** 탄창 재장전에 걸리는 시간 (초)
- **사정거리:** `bulletLifetime × bulletSpeed` 로 구현

### 5.5 고유 옵션 (무기별)

| 무기 | 고유 옵션 |
|:----|:---------|
| 💥 폭발 | 폭발 반경 증가 (1→5단계) |
| 🔫 속사 | 1회 발사 시 포탄 수 증가 (5→30발) |
| 🎯 저격 | 저격 거리 비례 데미지 계수 (2.50→1.55) |
| 🔩 관통 | 관통 데미지 감소율 감소 (3%→9% → 낮을수록 좋음) |

### 5.6 탄창 & 재장전 시스템
- 각 무기는 **독립적인 탄창** 보유
- **선택 중이지 않은 무기**는 재장전 시간마다 1발씩 자동 재장전
- 융합 무기 사용 시 **두 무기 모두 탄창 필요**, 발사 시 각각 1발 소모
- 한쪽 탄창이 비면 융합 발사 불가 → 단일 무기로 전환 필요

### 5.7 강화 시스템 (랜덤 가챠)
- **랜덤 무기 강화권** 사용 시:
  1. 무기 랜덤 선택 (동등 확률)
  2. 강화 옵션 랜덤 (데미지/탄창/재장전)
  3. 상승 단계 (1단계 85% / 2단계 10% / 3단계 5%)
- 최대 20단계, 3배수 단계는 상승치 2배
- 고유 옵션: 공용 옵션이 모두 5/10/15/20단계 도달 시 자동 상승

> 상세 수치 테이블(1~20단계)은 `07-사격수-무기-시스템.md` §10 참조.

## 7. 씬 & 게임 플로우

> 이 섹션은 v0.1에서 **신규 설계**된 부분입니다.
> mvp-v2의 단순 게임오버/리스타트를 확장하여 완전한 씬 시스템을 정의합니다.

### 7.1 씬 구조 개요

게임은 다음과 같은 씬(Scene)들로 구성됨:

```
┌─────────────┐
│  TITLE      │  ← 게임 실행 시 첫 화면
│  (메인 메뉴) │
└──────┬──────┘
       │ [Enter / Click]
       ▼
┌─────────────┐
│  PLAY       │  ← 실제 게임 플레이
│  (게임플레이) │
└──────┬──────┘
       │ [ESC]          │ [HP ≤ 0]         │ [승리 조건]
       ▼                 ▼                    ▼
┌─────────────┐   ┌─────────────┐    ┌─────────────┐
│  PAUSE      │   │  GAME OVER  │    │  VICTORY    │
│  (일시정지)  │   │  (게임오버)  │    │  (승리)      │
└──────┬──────┘   └──────┬──────┘    └──────┬──────┘
       │ [계속]          │ [R / 재시작]       │ [R / 재시작]
       ▼                 ▼                    ▼
  ┌─────────────┐   ┌─────────────┐    ┌─────────────┐
  │  PLAY (재개)  │   │  PLAY (처음) │    │  PLAY (처음) │
  └─────────────┘   └─────────────┘    └─────────────┘
```

### 7.2 씬별 상세

#### 7.2.1 TITLE (메인 메뉴) 🆕

| 요소 | 설명 |
|:----|:------|
| **배경** | 애니메이션 배경 (게임 월드 느낌의 무한 스크롤/파티클) |
| **타이틀** | 게임 로고/제목 — 중앙 상단 |
| **"Press Enter to Start"** | 하단 — 깜빡이는 텍스트 |
| **조작법 표시** | 화면 하단 또는 측면에 간단한 조작 가이드 |
| **버전 표시** | 우측 하단 — 현재 버전 (v0.1 등) |

**동작:**
- Enter 또는 마우스 클릭 → PLAY 씬으로 전환
- 페이드 인/아웃 전환 효과

**씬 데이터:**
```js
scene = {
  type: 'title',
  enterAt: timestamp,
  blinkTimer: 0,
}
```

#### 7.2.2 PLAY (게임 플레이) ✅ mvp-v2 확정 + 확장

mvp-v2의 게임 루프를 기반으로 하되, 다음이 추가됨:

| 기능 | 상태 | 설명 |
|:----|:----:|:------|
| 게임 루프 (requestAnimationFrame) | ✅ | mvp-v2 유지 |
| 탱크 조작/물리 | ✅ | §4 기반 |
| 무기 발사 + 탄창 | ✅ | §5 기반 (07 구현 완료) |
| 적 스폰/AI | ✅ | §6 기반 |
| 충돌/데미지 처리 | ✅ | mvp-v2 유지 |
| 게임오버 조건 (HP=0) | ✅ | mvp-v2 유지 |
| R키 리스타트 | ✅ | mvp-v2 유지 |
| **ESC 일시정지** | 🆕 | PAUSE 씬으로 전환 |
| **HUD** | 🆕 | §8 UI/HUD 참조 |
| **승리 조건** | 🆕 | 추후 게임 진행 시스템 완성 시 |

#### 7.2.3 PAUSE (일시정지) 🆕

| 요소 | 설명 |
|:----|:------|
| **오버레이** | 게임 화면 위 반투명 검정 오버레이 (게임 상태는 일시정지) |
| **"일시정지"** | 중앙 상단 텍스트 |
| **"ESC — 계속하기"** | 중앙 |
| **"R — 리스타트"** | 중앙 하단 (확인창 별도) |
| **배경** | 일시정지된 게임 화면이 흐릿하게 보임 |

**동작:**
- ESC 토글: PLAY ↔ PAUSE
- PAUSE 중 게임 루프는 정지 (requestAnimationFrame 중단 또는 freeze 플래그)
- R키: "정말 리스타트하시겠습니까?" 확인창 → 확인 시 리스타트 (타이틀로 또는 바로 플레이)

**일시정지 시 상태 처리:**
```js
gameState = {
  isPaused: true,
  pausedAt: timestamp,
  // 물리/타이머 모두 정지
}
```

#### 7.2.4 GAME OVER (게임 오버) ✅ mvp-v2 기반 확장

| 요소 | 설명 |
|:----|:------|
| **배경** | 게임 화면 위 붉은 톤 반투명 오버레이 (선택) 또는 정적 게임 화면 |
| **"GAME OVER"** | 중앙 큰 텍스트 (깜빡임 효과) |
| **결과 표시** | 생존 시간, 처치한 적 수, 획득 골드 등 |
| **"R — 다시 시작"** | 중앙 하단 |
| **"Enter — 메인 메뉴"** | 그 아래 |

**기존 mvp-v2 대비 확장:**
- 결과 통계 표시 (mvp-v2는 단순 GAME OVER 텍스트)
- 메인 메뉴로 돌아가기 (Enter) 옵션 추가
- 리스타트 시 부드러운 전환 (페이드)

**결과 통계 데이터:**
```js
gameResult = {
  survivalTime: 125.3,     // 초
  enemiesKilled: 47,
  weaponUsed: 'explosion', // 가장 많이 사용한 무기
  upgradeLevel: 3,          // 최대 달성 강화 단계
  score: 4700,              // 종합 점수 (추후 산식)
}
```

#### 7.2.5 VICTORY (승리) 🆕

| 요소 | 설명 |
|:----|:------|
| **배경** | 축하 효과 (파티클/빛 효과) — 추후 |
| **"VICTORY!"** | 중앙 큰 텍스트 (황금색) |
| **결과 표시** | GAME OVER과 동일한 통계 |
| **"R — 다시 시작"** | 중앙 하단 |
| **"Enter — 메인 메뉴"** | 그 아래 |

**조건:** 추후 게임 진행 시스템(웨이브/목표/보스) 완성 시 활성화.
현재 v0.1에서는 씬 구조만 정의하고 실제 전환 조건은 추후 구현.

### 7.3 씬 매니저 설계

씬 전환을 담당하는 SceneManager를 main.js에 추가:

```js
class SceneManager {
  constructor() {
    this.currentScene = 'title';   // 'title' | 'play' | 'pause' | 'gameover' | 'victory'
    this.previousScene = null;
    this.transitioning = false;     // 전환 중 플래그 (추후 페이드 효과)
    this.sceneData = {};            // 씬별 데이터 (타이머, 결과 등)
  }

  switchTo(sceneName, data) {
    this.previousScene = this.currentScene;
    this.currentScene = sceneName;
    this.sceneData[sceneName] = { ...data, enterAt: performance.now() };
    // 전환 효과: 페이드 아웃 → 전환 → 페이드 인
  }
}
```

**씬 전환 조건 요약:**

| 조건 | 현재 씬 | → | 대상 씬 |
|:----|:-------:|:---:|:-------:|
| 게임 실행 | — | → | TITLE |
| Enter / 클릭 | TITLE | → | PLAY |
| ESC (토글) | PLAY | → | PAUSE |
| ESC (토글) | PAUSE | → | PLAY |
| HP ≤ 0 | PLAY | → | GAME OVER |
| 승리 조건 달성 | PLAY | → | VICTORY |
| R | GAME OVER / VICTORY | → | PLAY (리스타트) |
| Enter | GAME OVER / VICTORY | → | TITLE |
