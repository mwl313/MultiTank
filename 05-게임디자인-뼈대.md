## 1. 게임 정체성

### 장르
2D 탑다운 스플릿 컨트롤 탱크 서바이벌 (뱀서라이크 파생)

### 코어 컨셉
**"같은 탱크, 다른 역할"**

두 명의 플레이어가 하나의 탱크를 조종한다.
- **운전수:** 탱크 이동, 회전, 드리프트, 램 어택, 아이템 수집
- **포수:** 포탑 조준, 발사, 화력 집중

서로의 역할이 완전히 다르기 때문에 **필연적인 소통**이 발생하고, 그것이 핵심 재미.

### 필수 사양
| 항목 | 결정 |
|:-----|:-----|
| 멀티플레이 | **2인 1조 온라인 협동 (기본 사양)** — WebSocket 등으로 구현 |
| 시점 | 탑다운 2D (카메라가 탱크 추적) |
| 조작 | 운전수: WASD, 포수: 마우스 조준+클릭 |
| 솔로 | MVP에서는 통합 조작으로 1인이 두 역할 수행 (멀티 스텁 유지) |
| 승리 조건 | 라운드 9 최종보스 격파 (~16분). 이후 무한 모드 |
| 패배 조건 | 탱크 HP 0 |

## 2. 3대 차별점

### ① 레벨업 2갈래 — 운전수 / 포수 독립 빌드

- **구조:** 운전수 4분기 × 5티어 / 포수 4분기 × 5티어
- **원칙:**
  - 운전수와 포수는 각자 자신의 역할에 해당하는 업그레이드만 선택 가능
  - 같은 분기를 연속 선택 시 시너지 보너스 (×1.1→×1.6)
  - Cross-branch(다른 분기 섞기)도 허용하나 시너지 없음 → "선택의 대가"
  - 세이프존 내에서 Tab키로 각자 독립 UI

### ② 운전수도 어택 — "운전수는 전사다"

운전수는 단순한 도망·이동 담당이 아니라 직접 전투에 기여하는 공격적 역할:
- **드리프트 슬라이딩:** 고속 선회 시 탱크 측면에 충돌한 적에게 데미지 (MVP 구현 완료)
- **돌진:** 적 방향으로 순간 가속 + 경로 적 넉백
- **반사 장갑:** 피격 시 적에게 데미지 반사
- **충격파:** 궁극기 광란 질주 — 적진을 쓸어버리는 카타르시스

### ③ 전장 이동이 게임플레이다 — "전진과 귀환"

기존 뱀서라이크의 정적 버티기와 달리, 탱크가 거대한 전장을 누비며 목표를 클리어한다.

**핵심 메커니즘: 세이프존 (Safe Zone)**
- 맵 곳곳에 배치된 안전 구역 (4개, 순차 개방)
- 진입 시: 적 추적 중단·배회 전환, HP 천천히 회복
- 진입 중: Tab키로 스킬트리 업그레이드, 룰렛 사용 가능
- 머물기 제한: 30초 초과 시 적이 점점 접근 → "영원히 숨을 순 없다"

**전장-세이프존 리듬:**
전투(위험) → 세이프존(정비·성장) → 다음 목표로 진격 → 반복
→ 운전수의 기동 능력이 "다음 세이프존까지 얼마나 빨리 갈 수 있는가"라는 전략 자원

## 3. 멀티플레이어 아키텍처 (개요)

### 기본 구조
- **서버-클라이언트 모델:** Node.js + WebSocket
- **방 시스템:** 호스트가 방 생성 → 코드 공유 → 게스트 참가
- **역할 할당:** 방 입장 시 운전수/포수 선택

### 동기화 대상
| 데이터 | 전송 방향 | 주기 |
|:------|:---------|:-----|
| 운전수 입력 (WASD, 액션) | 클라이언트 → 서버 | 매 프레임 |
| 포수 입력 (조준각, 발사) | 클라이언트 → 서버 | 매 프레임 |
| 탱크 상태 (위치, HP, 속도) | 서버 → 클라이언트 | 20Hz |
| 적 상태 (위치, HP) | 서버 → 클라이언트 | 20Hz |
| 세이프존·라운드 상태 | 서버 → 클라이언트 | 이벤트 발생 시 |

### 권한 모델
- **서버 권위:** 게임 로직(충돌, HP, 스폰, 라운드)은 서버가 처리
- 세이프존 판정은 서버 사이드 → 두 클라이언트 동일한 세이프존 상태 공유

## 4. 스킬트리 구조 — 4분기 × 5티어

### 4-0. 티어 구조

```
T1 ── 입문 (1옵션)             → "나는 이 길이다"
T2 ── 갈림길 (2옵션 중 택1)     → "A냐 B냐"
T3 ── 심화 (4옵션 중 2개만 활성화, T2 선택에 따라) → "더 깊게"
T4 ── 자유 선택 (2옵션)         → "지금 필요한 건"
T5 ── 궁극기 (1옵션, T2-T3 경로에 따라 변형) → "끝이다!"
```

**빌드 다양성:** 한 분기당 T2(2) × T3(2) × T4(2) = 8가지 경로.

### 4-1. 포수 4분기

#### 🔴 파괴자 (Demolisher) — 한 방이다. 맞으면 없어진다.

## 5. 경제 & 아이템 시스템 (뼈대)

### 5-1. 재화 체계 (3종)

| 재화 | 획득 방법 | 용도 |
|:----|:---------|:-----|
| **업그레이드 재화** [이름 미정] | 적 처치 시 드롭 | 스킬트리 티어 해금·강화 |
| **룰렛 재화** [이름 미정] | 적 처치 시 확률 드롭 | 룰렛을 돌려 유물 획득 |
| **골드** | 적 처치 시 자동 획득 | 웨이브 간 보급품 구매, 리롤 등 |

### 5-2. 룰렛 시스템 (유물)

```
[룰렛 재화]가 일정량 모이면 → 특정 키를 눌러 룰렛 발동

룰렛 결과:
  일반 유물 (고확률): 경미한 패시브 효과
  레어 유물 (저확률): 강력한 패시브 효과
  초레어 유물 (극저확률): 게임 체인저급 효과

룰렛은 세이프존 안에서만 사용 가능.
룰렛 재화가 쌓인 걸 보면서 "세이프존 가서 돌려야지" → 도파민
```

#### 유물 설계 방향 (추후)
- 유물은 획득 즉시 발동되는 판 전체 지속 패시브
- 최대 보유 개수 제한 (2~3개, 초과 시 교체 선택)
- 유물 간 시너지 또는 상호작용 가능

#### 3D 기획서와의 차이
| 3D 기획서 | 2D 변경 |
|:---------|:--------|
| 보스 드롭으로 유물 획득 (수동) | 룰렛으로 능동적 획득 |
| 확률 0.1~3%의 수동적 운 | 언제 돌릴지 플레이어가 결정 → 전략적 선택 |
| 맵에 떨어진 아이템 주워야 함 | UI 룰렛 → 즉시 획득 |

### 5-3. 드롭 아이템 (3종)

| 카테고리 | 획득 방식 | 용도 |
|:--------|:---------|:-----|
| **업그레이드 재화** | 적 처치 시 드롭 | 스킬트리 티어 해금 |
| **룰렛 재화** | 적 처치 시 확률 드롭 | 룰렛 1회 |
| **전장 보급품** | 적 처치 시 확률 드롭 → 수동 획득(탱크로 밟기) | 시간제 버프·회복·유틸(Megabonk 참고) |

#### 설계 원칙
- 드롭 아이템은 스킬트리 강화와 연계 — 적 처치 → 재화 → 성장 선순환
- 전장 보급품은 순간적 전투 보조
- 룰렛 재화는 "돌릴까? 더 모을까?" 긴장감
