# Claude Code + DeepSeek — Windows 코딩 위임 로드맵

> 작업 플로우: **Aria(Mac)가 파일 준비 → GitHub Push → 주인님(Windows)이 Pull → Claude Code 실행**
> Claude Code 모델: `ANTHROPIC_MODEL=deepseek-v4-pro` (설계) + `CLAUDE_CODE_SUBAGENT_MODEL=deepseek-v4-flash` (구현)

---

## 전체 흐름

```
Aria(Mac):
  1. config 파일 생성
  2. spec 문서 업데이트
  3. GitHub Commit + Push
  ──────────────────────────────
주인님(Windows):
  4. git pull
  5. Claude Code 실행 (Phase별 명령어 복붙)
  6. 결과 확인 → 필요시 Aria에게 피드백 ↔ 반복
```

## 사전 준비 (Windows - 1회)

### 환경변수 설정

Claude Code가 Windows에서 DeepSeek API를 쓰려면 환경변수 필요:

```powershell
# PowerShell (관리자 모드) 또는 사용자 환경변수에 추가
$env:ANTHROPIC_MODEL = "deepseek-v4-pro"
$env:CLAUDE_CODE_SUBAGENT_MODEL = "deepseek-v4-flash"
$env:CLAUDE_CODE_EFFORT_LEVEL = "max"
$env:ANTHROPIC_AUTH_TOKEN = "여기에_DeepSeek_API_키"
```

또는 `.claude.env` 파일을 프로젝트 루트에 만들어도 됨:

```env
ANTHROPIC_MODEL=deepseek-v4-pro
CLAUDE_CODE_SUBAGENT_MODEL=deepseek-v4-flash
CLAUDE_CODE_EFFORT_LEVEL=max
ANTHROPIC_AUTH_TOKEN=여기에_DeepSeek_API_키
```

## Windows에서의 추가 고려사항

### 1. 경로 문제

- Claude Code가 생성하는 import 경로가 `\`(백슬래시)가 아니라 `/`(슬래시)인지 확인
- ES6 module `type="module"`은 `file://` 프로토콜에서 CORS 문제 발생 가능
  → 해결: **로컬 서버 띄우기**

### 2. Windows 로컬 서버 (필수)

ES6 모듈은 `file://`에서 CORS 에러. 간단한 HTTP 서버 필요:

```powershell
# 방법 1: Node.js (권장)
npx http-server . -p 8080 -c-1

# 방법 2: Python
python -m http.server 8080

# 방법 3: PowerShell (내장)
powershell -Command "Start-Process 'http://localhost:8080'"
```

브라우저 → `http://localhost:8080` 접속해서 테스트.

### 3. Git

```powershell
git pull origin main   # Aria가 푸시한 config + 명령어 파일 받기
claude -p "...phase1..."  # 코딩 실행
git add -A && git commit -m "Phase 1 완료"
git push origin main   # Aria가 검증할 수 있도록
```

## Phase 명령어 파일 생성 순서 (Aria)

Phase 1~6의 명령어 파일(`mvp-phase*.txt`)은 **지금 당장 다 만들 필요 없음.**
각 Phase가 완료되고 나서 다음 Phase 명령어를 Aria가 생성 + GitHub Push.

즉:
```
Phase 1 완료 (주인님) → git push
→ Aria가 결과 확인 → Phase 2 명령어 파일 생성 → git push
→ 주인님이 git pull → Phase 2 실행
...반복...
```

이렇게 하면 각 Phase의 결과물을 보고 다음 명령어를 더 정확하게 만들 수 있어요.
