## 1. 개요

사격수는 무기를 선택해 공격하며, 각 무기는 종류별로 특성을 지니고 있음.
이 문서는 기존 게임 디자인 뼈대(`05`)의 **포수(Gunner) 역할**을 구체화하며,
포수가 사용하는 4종 무기의 능력치, 강화, 조작, 융합 시스템을 정의함.

> 🔗 기존 설계와의 관계: 기존 포수 4분기(파괴자·속사포·저격수·교란자) 스킬트리는
> 이 무기 시스템 위에서 동작하도록 재구성됨. 무기 선택이 선행되고, 스킬트리는 선택된 무기를 강화/변형함.

## 2. 무기 종류

무기는 **4종류**로 구성됨. 플레이어는 **단일 무기**와 **융합 무기**를 사용 가능.

### 2.1 단일 무기 (4종)

| 번호 | 이름 | 아이콘 | 코어 컨셉 |
|:----:|:-----|:------:|:---------|
| 1 | **폭발** (Explosion) | 💥 | 탄착 지점에 넓은 범위 피해 |
| 2 | **속사** (Rapid Fire) | 🔫 | 1회 발사마다 여러 발의 포탄을 발사 |
| 3 | **저격** (Sniper) | 🎯 | 더 먼 거리까지 사격 가능 |
| 4 | **관통** (Piercing) | 🔩 | 포탄이 적 또는 지형을 관통하여 진행 |

### 2.2 융합 무기 (6개 조합)

두 개의 단일 무기를 동시에 사용하는 것을 의미. 총 6개 조합 존재.

| 조합 | 코드 |
|:----|:-----|
| 폭발 + 속사 | `1+2` |
| 폭발 + 저격 | `1+3` |
| 폭발 + 관통 | `1+4` |
| 속사 + 저격 | `2+3` |
| 속사 + 관통 | `2+4` |
| 저격 + 관통 | `3+4` |

융합 무기 선택 시 두 무기의 포탄을 모두 소모하며, 데미지 규칙은 §9 참조.

## 3. 무기 옵션

### 3.1 공용 옵션 (모든 무기 공통)

| 옵션 | 설명 | 단위 |
|:----|:-----|:----:|
| **데미지** | 적에게 입히는 피해 수치 | 수치 |
| **탄창 수** | 하나의 탄창 내에 들어가는 포탄의 수 | 발 |
| **재장전 시간** | 포탄 탄창을 재장전하는데 걸리는 시간 | 초 |
| **사정거리** | 포탄이 발사하여 도달 가능한 최대 거리 | px |

> 사정거리는 현재 게임의 `bulletLifetime × bulletSpeed`로 구현함.
> 무기별 `bulletLifetime` 값을 조정하여 사정거리를 표현.

### 3.2 고유 옵션 (무기별)

| 무기 | 고유 옵션 | 설명 |
|:----|:---------|:-----|
| 💥 폭발 | **폭발 데미지 반경 증가** | 폭발 범위가 단계에 따라 확대됨 |
| 🔫 속사 | **연속 발사 횟수 증가** | 1회 발사 시 발사하는 포탄 수 증가 |
| 🎯 저격 | **저격 거리 비례 데미지 증가** | 먼 거리 명중 시 추가 데미지 계수 적용 |
| 🔩 관통 | **관통 시 피해 감소율 감소** | 관통할 때마다 감소하는 데미지 비율 조정 |

## 4. 무기 선택 및 발사

### 4.1 무기 선택 (숫자키)

| 조작 | 결과 |
|:----|:-----|
| 숫자키 **1** | 폭발 무기 선택 |
| 숫자키 **2** | 속사 무기 선택 |
| 숫자키 **3** | 저격 무기 선택 |
| 숫자키 **4** | 관통 무기 선택 |
| 숫자키 **1 + 2** 동시 | 융합: 폭발+속사 |
| 숫자키 **1 + 3** 동시 | 융합: 폭발+저격 |
| 숫자키 **1 + 4** 동시 | 융합: 폭발+관통 |
| 숫자키 **2 + 3** 동시 | 융합: 속사+저격 |
| 숫자키 **2 + 4** 동시 | 융합: 속사+관통 |
| 숫자키 **3 + 4** 동시 | 융합: 저격+관통 |

- 하나만 누를 경우 = 단일 무기
- 두 개를 동시에 누를 경우 = 융합 무기
- 융합 무기 선택 시, 사용 중인 모든 무기에 장전된 포탄이 있어야 발사 가능 (§6 참조)

### 4.2 발사

- 무기 선택 후 마우스 포인터로 조준
- 마우스 **좌클릭** 눌러 발사
- 쿨다운(`fireInterval`) 적용

### 4.3 기존 input.js 확장

기존 `PLAYER_INPUT.gunner`에 무기 선택 상태를 추가:

```js
// 새 필드
gunner.selectedWeapons: Set<number>  // {1}, {2}, {1,2} 등
```

키보드 이벤트 '1'~'4'를 감지하여 토글.
"동시 누름" 판정은 `keydown` 타임스탬프 간격 100ms 이내로 판단.

## 5. 데미지 계산 공식

### 5.1 기본 데미지 공식

```
(적에게 입히는 피해) = (무기 데미지) - (적 방어력)
```

- 방어력은 무기 데미지 계산이 끝난 이후에 적용
- 즉, 먼저 무기의 강화된 데미지 수치로 계산 → 그 결과에서 적 방어력 차감

### 5.2 방어력 처리

- 현재 `enemy.js`의 `enemy` 객체에 `defense` 필드 추가
- `collision.js`에서 데미지 계산 시 차감:
  ```js
  const finalDamage = Math.max(1, bullet.damage - enemy.defense);
  enemy.hp -= finalDamage;
  ```
- 최소 데미지 1 보장

## 6. 탄창 및 재장전 시스템

### 6.1 기본 규칙

- 각 무기는 독립적인 탄창을 가짐
- 탄창에 장전된 포탄이 없는 무기는 발사할 수 없음
- 융합 무기 사용 시: 사용 중인 **모든 무기에 장전된 포탄이 있어야 함**
- 융합 무기 발사 시: 사용 중인 **모든 무기의 포탄이 1발씩 소모됨**

### 6.2 자동 재장전

- **현재 사용 중이지 않은(=선택하지 않은) 무기**는 재장전 시간마다 **1발씩 자동으로 재장전**됨
- 최대 탄창 수 이상으로는 재장전되지 않음
- 예: 폭발(탄창3)을 선택 중이면, 속사·저격·관통은 각각의 재장전 시간마다 1발씩 자동 충전

### 6.3 구현 방향

```js
// 각 무기의 탄창 상태
const weaponMagazines = {
  1: { current: 3, max: 3, reloadTimer: 0, reloadInterval: 5.0 },  // 폭발
  2: { current: 3, max: 3, reloadTimer: 0, reloadInterval: 3.0 },  // 속사
  3: { current: 1, max: 1, reloadTimer: 0, reloadInterval: 10.0 }, // 저격
  4: { current: 30, max: 30, reloadTimer: 0, reloadInterval: 2.0 }, // 관통
};

// 선택되지 않은 무기만 자동 재장전
// 선택한 무기의 탄창만 발사 시 소모
```

## 7. 강화 시스템

### 7.1 개요 — 랜덤 가챠

**랜덤 무기 강화권**을 사용하면:
1. 랜덤한 무기
2. 랜덤한 옵션
3. 랜덤 수치만큼

위 순서대로 강화됨.

### 7.2 결정 순서

```
강화권 사용
  → 1. 무기 종류 결정 (모든 무기 동등 확률)
  → 2. 강화 옵션 결정 (해당 무기의 모든 강화 가능 옵션 중 랜덤)
  → 3. 강화 단계 결정 (1~3단계, 확률 테이블)
```

### 7.3 제외 조건

- 특정 무기에서 **최대 수치(20단계)까지 강화된 옵션**은 가챠 선택지에서 제외
- **모든 옵션이 최대 강화된 무기**는 가챠 선택지에서 제외
- **모든 무기의 모든 옵션**이 최대로 강화된 경우 무기 강화권을 사용할 수 없음

### 7.4 공용 옵션 강화

랜덤 무기 강화권 사용 시, 아래 **공용 옵션** 중 하나가 랜덤으로 강화됨:

- 데미지
- 탄창 수
- 재장전 시간

**단계 구조:**
- 모든 옵션은 게임 시작 시 **1단계**로 시작
- 최대 **20단계**까지 강화 가능
- **3의 배수 단계**에서는 옵션 상승 수치가 **두 배**로 적용됨

### 7.5 강화 단계 확률

| 상승 단계 | 확률 |
|:---------:|:----:|
| 1단계 | 85% |
| 2단계 | 10% |
| 3단계 | 5% |

### 7.6 고유 옵션 강화

고유 옵션은 **각 무기의 모든 공용 옵션이 5의 배수가 넘어갈 때마다** 자동으로 강화됨.

> 예: 폭발 무기의 데미지, 탄창 수, 재장전 시간이 모두 5단계에 도달하면
> 폭발 반경이 1단계에서 2단계로 자동 강화됨.

**고유 옵션 단계표:**

| 조건 (모든 공용옵션 ≥) | 고유 옵션 단계 |
|:---------------------:|:--------------:|
| 5단계 | 2단계 |
| 10단계 | 3단계 |
| 15단계 | 4단계 |
| 20단계 | 5단계 |

## 8. 무기별 상세 동작

### 8.1 💥 폭발 (Explosion)

- 발사 시 **최종 탄착 지점에서 폭발**하여 범위 피해를 입힘
- 최종 탄착 지점은 마우스 포인터의 위치를 목표로 함
- 중간에 **장애물 또는 적과 충돌**할 경우 해당 지점이 최종 탄착 지점이 됨
- '관통'과 융합 무기로 사용하는 경우:
  - 적을 관통하는 경우에는 폭발이 일어나지 않음
  - 최종 탄착 지점에서만 폭발함

### 8.2 🔫 속사 (Rapid Fire)

- 1회 발사 시 **발사 횟수만큼의 포탄**을 발사함
- 각 포탄은 약간의 산개각(5~10°)을 가짐
- 한 번의 클릭으로 `발사 횟수`만큼의 총알을 순차/동시 발사

### 8.3 🎯 저격 (Sniper)

- 다른 무기들에 비해 **3배 긴 사거리**를 가짐
- 다른 무기 사거리 이상의 사거리는 **조준선이 다른 UI**로 표시됨 (예: 얇은 십자선 → 굵은 원형 조준점)
- 다른 무기 사거리 이상에서 공격하는 경우 **'원거리 저격'**으로 간주
- 원거리 저격 시 **저격 거리 비례 데미지 계수**를 적용
- 융합 무기로 사용 시에도 원거리 저격 데미지 계수가 적용될 수 있음 (§9 참조)

### 8.4 🔩 관통 (Piercing)

- **적 또는 지형을 관통**하여 타격 가능
- 관통할 때마다 데미지가 감소함
  - 예: 기본 데미지 100에서 데미지 감소율이 20%인 경우
    - 첫 관통 시 데미지 80
    - 두번째 관통 시 데미지 64
- 공식: `관통 데미지 = 원본 데미지 × (1 - 감소율)^(관통 횟수)`

## 9. 융합 무기 시스템

### 9.1 탄창 규칙

- 융합 무기 사용 중인 **모든 무기에 장전된 포탄이 있어야 함**
- 융합 무기 발사 시 **모든 무기의 포탄이 1발씩 소모됨**
- 한쪽 탄창이 비면 융합 무기 발사 불가 → 단일 무기로 전환 필요

### 9.2 융합별 데미지 규칙

| 융합 조합 | 데미지 적용 규칙 |
|:----------|:----------------|
| **폭발 + 속사** | 폭발 데미지를 적용 (속사는 발사 수만 영향) |
| **폭발 + 저격** | 폭발 데미지 적용. '원거리 저격' 조건 만족 시 저격 데미지 계수 반영 |
| **폭발 + 관통** | 적 관통 시 관통 데미지. 최종 탄착지 폭발 시 폭발 데미지 적용 |
| **속사 + 저격** | 속사와 저격 중 **높은 쪽 데미지** 적용. 원거리 저격 조건 시 저격 계수 반영 |
| **속사 + 관통** | 속사와 관통 중 **높은 쪽 데미지** 적용 |
| **저격 + 관통** | 저격과 관통 중 **높은 쪽 데미지** 적용. 원거리 저격 조건 시 저격 계수 반영 |

> 융합 발사는 포탄 1발을 발사하되, 두 무기의 특성을 결합한 효과를 냄.
> 예: 폭발+속사 = 폭발 포탄 1발을 발사하지만 속사의 발사 속도 간격을 따름.

### 9.3 융합 무기 발사 간격

- 융합 무기의 발사 간격(`fireInterval`)은 **두 무기 중 더 긴 쪽**을 따름
- 단, '속사'가 포함된 조합은 속사의 `fireInterval`을 우선 적용

## 10. 무기 옵션 테이블 (1~20단계)

모든 무기는 1단계(게임 시작)에서 시작하며,
랜덤 강화권을 통해 최대 20단계까지 강화 가능.

| 7 | 14 | 2.70 | 13 | 2 |
| 8 | 16 | 2.65 | 14 | 2 |
| 9 | 18 | 2.60 | 16 | 2 |
| 10 | 20 | 2.55 | 17 | **3** |
| 11 | 22 | 2.50 | 18 | 3 |
| 12 | 24 | 2.45 | 20 | 3 |

### 11.7 tryFire 함수 확장 (tank.js)

기존 단일 `tryFire`를 무기 선택 기반 다중 발사로 확장:

```js
export function tryFire(tank, input, bulletPool, magazineSystem, upgradeSystem) {
  if (!input.gunner.fire) return;
  if (tank.fireCooldown > 0) return;

  const selected = input.gunner.selectedWeapons;
  if (selected.size === 0) return;

  // 단일 무기
  if (selected.size === 1) {
    const weaponId = [...selected][0];
    // 탄창 확인 & 소모
    if (!magazineSystem.canConsume(weaponId)) return;
    magazineSystem.consume(weaponId);
    fireWeapon(tank, weaponId, bulletPool, upgradeSystem);
  }
  // 융합 무기
  else if (selected.size === 2) {
    const [id1, id2] = [...selected];
    // 두 무기 모두 탄창 확인
    if (!magazineSystem.canConsumeBoth(id1, id2)) return;
    magazineSystem.consumeBoth(id1, id2);
    fireFusionWeapon(tank, id1, id2, bulletPool, upgradeSystem);
  }

  tank.fireCooldown = getFireInterval(selected, upgradeSystem);
}
```
