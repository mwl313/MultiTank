> 작성일: 2026-07-18
> 목적: 2D 탑다운 스플릿 컨트롤 게임의 최소 실행 가능 제품 정의

## MVP 철학

**"2D 탑다운 + 스플릿 컨트롤이 재미있는가?"** 를 검증하는 데 집중.

## 기술 스택 제안

| 계층 | 기술 | 사유 |
|:----|:-----|:------|
| **렌더링** | **Canvas 2D (Vanilla JS)** | 2D 탑다운에는 Three.js가 오버킬. 움직이는 물체 많아도 Canvas 2D로 충분 |
| **게임 루프** | requestAnimationFrame + 고정 틱 | 직접 구현, socket.io 제거 (MVP는 로컬 싱글 전용) |
| **충돌** | 사각형 충돌체 (AABB) | 복잡한 Octree 불필요. 사각형-사각형, 원-사각형만 있으면 됨 |
| **언어** | JavaScript (ES6+) | 기존 코드베이스와 동일, 빠른 프로토타입 |
| **빌드 도구** | 없음 (생략) | Vite 등은 검증 이후 도입 |

### 대안: Phaser 3
- 장점: 게임 엔진 내장 (씬 관리, 충돌, 입력, 카메라, 파티클...)
- 단점: 학습 비용, 프레임워크에 갇힘
- **결론: MVP는 Vanilla Canvas 2D. Phaser는 확장 단계에서 고려.**

## MVP 파일 구조

```
nhn-ai-game-jam-2026/
└── 2D리메이크/
    ├── mvp/
    │   ├── index.html         ← 단일 페이지, Canvas + CSS + UI
    │   ├── game.js            ← 게임 루프, 상태 관리, 웨이브
    │   ├── tank.js            ← 탱크 (운전수+포수 통합, split input 분기)
    │   ├── enemy.js           ← 적 스폰, AI 이동, 충돌
    │   ├── bullet.js          ← 총알 생성, 이동, 충돌
    │   └── upgrades.js        ← 레벨업 UI, 운전수/포수 업그레이드 풀
    └── mvp-ready.md           ← MVP 완료 체크리스트
```

### 대안: 싱글 파일
테스트 속도를 위해 처음에는 `index.html` 하나로 시작하고, 검증 후 분리해도 됨.

## MVP 상세 구현 명세

### 1. 게임 상태
```
LOADING → PLAYING → LEVELUP → PLAYING → ... → GAMEOVER
```

### 2. 탱크 (tank.js)

| 속성 | 초기값 |
|:----|:-------|
| 위치 (x, y) | 맵 중앙 |
| 크기 | 60×40 px (회전 가능한 사각형) |
| 이동속도 | 200 px/s (운전수 업글 시 증가) |
| HP | 100 |
| 회전 | 마우스 커서 방향으로 즉시 |
| 포탑 각도 | 마우스 커서 - 탱크 중심 벡터 |
| 발사 간격 | 500ms (포수 업글 시 감소) |
| 총알 데미지 | 10 |
| 충돌 데미지 (람) | 5 + (속도 계수) |

**입력 분기:**
- 솔로 모드: 플레이어가 운전수 + 포수 모두 조작
  - WASD = 이동
  - 마우스 = 포탑 조준
  - 클릭 = 발사
  - Space = 드리프트/회피 (액션키)

### 3. 적 (enemy.js)

| 타입 | 색상 | HP | 속도 | 데미지 | 크기 |
|:----|:----|:---|:-----|:------|:----|
| 기본병 (Scout) | 빨강 | 20 | 120 px/s | 5 | ⌀20px |
| 전사 (Bruiser) | 주황 | 60 | 80 px/s | 10 | ⌀30px |
| 신속형 (Runner) | 노랑 | 10 | 200 px/s | 3 | ⌀15px |

**웨이브 증가 공식:**
- 1웨이브: 기본병 5마리
- 웨이브당 +2마리 + 신속형 등장 (3웨이브부터) + 전사 등장 (5웨이브부터)
- 60초마다 웨이브 상승

**적 AI:**
- 탱크 방향으로 일정한 속도로 접근
- 적끼리는 밀어내기 (겹침 방지, 기본적인 분산)
- 탱크와 접촉 시 데미지 + 짧은 경직

### 4. 총알 (bullet.js)

| 속성 | 값 |
|:----|:----|
| 속도 | 600 px/s |
| 크기 | ⌀6px |
| 수명 | 1.5초 (거리 900px) |
| 장애물 충돌 | 장애물에 맞으면 소멸 |
| 적 충돌 | 데미지 전달 후 소멸 (관통 업글 시 예외) |

### 5. 레벨업 시스템 (upgrades.js)

**경험치:** 적 사망 시 일정 EXP 드롭. 필요 EXP = 50 + (레벨 × 20)

**레벨업 시:** 화면 중앙에 2개 카드 표시
- 좌측 = 운전수 업그레이드 2지선다
- 우측 = 포수 업그레이드 2지선다
- 게임 일시정지, 선택하면 재개

#### 운전수 업그레이드 풀
| 이름 | 설명 | 최대 중첩 |
|:----|:-----|:---------|
| 엔진 부스트 | 이동속도 +12% | 5 |
| 장갑 강화 | 받는 피해 -10% | 3 |
| 드리프트 대미지 | 램 데미지 +50% | 3 |
| 스테미나 | 회피 쿨다운 -20% | 3 |
| 충돌 저항 | 적 충돌 경직 시간 감소 | 2 |

#### 포수 업그레이드 풀
| 이름 | 설명 | 최대 중첩 |
|:----|:-----|:---------|
| 속사 | 발사 간격 -12% | 5 |
| 강탄 | 총알 데미지 +20% | 3 |
| 확산 | 총알 2발 발사 (산개각 15°) | 2 |
| 장전술 | 탄창 +1 (연속 발사) | 3 |
| 관통 | 총알이 적 1명 관통 | 1 |

### 6. 미션: 서바이벌
- 조건: 정해진 시간 동안 생존
  - 1스테이지: 90초 생존 → Victory
  - Victory 후 일시정지 없이 바로 "어디로 이동할까?" (다음 전장 선택)
- MVP는 1스테이지만 구현 (90초 버티기)

### 7. UI (HTML/CSS 오버레이)

| 요소 | 위치 | 내용 |
|:----|:----|:-----|
| HP 바 | 좌측 상단 | 탱크 HP 그래픽 바 |
| 웨이브 표시 | 우측 상단 | "Wave 3" |
| 타이머 | 상단 중앙 | 남은 시간 (서바이벌) |
| 레벨/EXP | HP 바 아래 | "Driver Lv.3 | Gunner Lv.2" |
| 업그레이드 카드 | 화면 중앙 (팝업) | 좌: 운전수 2지선다 / 우: 포수 2지선다 |
| 게임오버 | 화면 중앙 | "GAME OVER - R to Restart" |
